## 学び・気づき

- GitHub Pages のソースブランチ設定は盲点になりやすい。
  `gh-pages` ブランチが残っていると `main` に push しても反映されない。
  → 次回新規リポジトリを作るときは最初から `main` を指定する

- インドオーナー（素人）への説明で有効だったフレーム：
  「MUJIに投資 = 劣化しない資産を買うこと」
  「流行インテリアは3年で償却が終わるが、MUJIは10年後も資産として残る」
  → 財務数値より「例え話」の方が腹落ちしやすい

- デザイン哲学の話は「なぜ長期投資として合理的か」の文脈で話すと
  素人にも刺さる。抽象論より「リノベ代がかからない」という具体に繋げる
